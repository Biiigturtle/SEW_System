# SEW 발주 검수 시스템

바코드 스캐너로 발주 검수, 전표 처리, 다중 스캐너 합산을 지원하는 웹 기반 시스템입니다.
별도 설치 없이 브라우저에서 바로 실행됩니다.

---

## 구성 파일

-------------------------------------------------------------
| `발주검수_SEW_v2.html` | 바코드 스캔으로 발주 대비 입고 수량 검수      |
| `전표처리_SEW.html`    | 바코드 스캔으로 이지어드민 전표처리 데이터 생성 |
| `스캔합산_SEW.html`    | 두 컴퓨터의 스캔 결과를 합산하여 발주와 비교   |
-------------------------------------------------------------

## 사전 준비

### 필요한 파일
- **페이히어 재고 DB** (`상품정보.xlsx`) — 바코드 → 상품명·사이즈 매핑
- **이지어드민 발주 DB** (`pack_data*.xls`) — 이번 발주 내역
- **현재고조회 파일** (`현재고조회*.xls`) — 전표처리 시 상품코드 조회용

### 권장 환경
- 스캐너: USB 유선 스캐너 권장 (블루투스 사용 시 아래 주의사항 참고)
- 와이파이: **5GHz 대역** 연결 권장 (블루투스와 2.4GHz는 같은 주파수 대역으로 간섭 발생 가능)

## 발주 검수 사용법

### 1. 시작
1. `발주검수_SEW_v2.html` 실행
2. **페이히어 재고 DB** 업로드 — 바코드·상품명 자동 인식
3. **이지어드민 발주 DB** 업로드 — `pack_data*.xls` 자동 인식 (컬럼 선택 불필요)
4. **검수 시작** 클릭

> 두 파일 모두 선택사항입니다. 발주 파일 없이 재고 카운팅 용도로도 사용 가능합니다.

### 2. 스캔
- 바코드 스캐너로 스캔하거나 직접 입력 후 Enter
- 결과는 스캔 즉시 화면에 표시되고 소리로 구분됩니다
- 스캔 입력 시 언어가 영어로 되어있는지 확인

---------------------------------
| 맑은 단음 (삑)       |    정상 스캔 |
| 내려가는 2음 (뚝↘뚝)  |   미발주 상품 |
| 거친 2연타 (부웅·부웅) | 미등록 바코드 |
----------------------------------

### 3. 수기 조정
스캔 수량을 수기로 수정해야 할 경우 테이블의 **수기조정** 컬럼에서 `+` / `−` 버튼으로 조정합니다.

### 4. 미등록 바코드 처리
1. 미등록 바코드 스캔 시 경고음과 함께 스캔 로그에 기록
2. 로그 항목 옆 **등록** 버튼 클릭
3. 상품명·사이즈 입력 후 저장
4. 등록 즉시 해당 바코드의 기존 미등록 스캔 전부 합산 처리
5. 등록 정보는 Firebase에 저장되어 **모든 컴퓨터에서 자동 공유**

> 등록 정보 수정: 화면 하단 **공유 등록 바코드** 패널에서 **수정** 버튼 클릭

### 5. 내보내기

--------------------------------------------
| CSV         |                 검수 결과 전체 |
| 합산용 내보내기 | 스캔합산 앱에 사용할 JSON 파일    |
| 신규등록 CSV  | 이번 세션에 수기 등록한 바코드 목록  |
--------------------------------------------

> **합산용 내보내기는 미등록 바코드를 등록한 뒤에 하세요.**
> 등록하지 않은 미등록 바코드는 합산에서 제외됩니다.

---

## 전표 처리 사용법

### 1. 시작
1. `전표처리_SEW.html` 실행
2. **상품정보 파일** (페이히어 재고 DB) 업로드
3. **현재고조회 파일** 업로드
4. **전표 처리 시작** 클릭

### 2. 스캔
- 바코드 스캔 시 상품명 조회 → 상품코드 자동 매칭
- **전표처리 목록** 탭: 매칭 성공 항목 (이지어드민 입력용)
- **매칭 실패** 탭: 상품코드 미매칭 항목 — 확인 후 내보내기

### 3. 내보내기
- **전표 XLSX**: 이지어드민 전표처리 시트 형식으로 바로 사용 가능

## 스캔 합산 사용법 (스캐너 2대 운영 시)

### 운영 방식
스캐너 2대를 각각 다른 컴퓨터에 연결하여 발주 물량을 나눠서 스캔한 뒤 합산합니다.

```
컴퓨터 A (스캐너 A) ─┐
                      ├─→ 스캔합산 앱 → 발주 비교
컴퓨터 B (스캐너 B) ─┘
```

### 절차

**각 컴퓨터에서:**
1. `발주검수_SEW_v2.html` 실행
2. 페이히어 재고 DB + 이지어드민 발주 DB 업로드
3. 담당 물량 스캔
4. 미등록 바코드가 있으면 **등록 완료 후** 내보내기
5. **합산용 내보내기** → `스캔데이터_날짜.json` 저장

**합산 컴퓨터에서:**
1. `스캔합산_SEW.html` 실행
2. **이지어드민 발주 DB** 업로드
3. **컴퓨터 A 스캔** + **컴퓨터 B 스캔** JSON 파일 업로드
4. **합산 실행**
5. 결과 확인 후 XLSX 내보내기

> A 또는 B 한 파일만 올려도 합산 실행 가능합니다.

## 공유 등록 바코드 (Firebase)

페이히어 재고 DB에 없는 신규 상품을 등록하면 Firebase에 저장되어 모든 컴퓨터에서 자동으로 공유됩니다.

- **오프라인 시**: 로컬에 임시 저장 → 인터넷 연결 시 자동 전송
- **연결 상태**: 헤더에서 실시간 확인 가능 (초록: 연결됨 / 노랑: 오프라인)
- **수정/삭제**: 화면 하단 **공유 등록 바코드** 패널 또는 Firebase 콘솔에서 직접 가능
- Firebase 콘솔: [console.firebase.google.com](https://console.firebase.google.com) → SEW SYSTEM → Realtime Database

> Firebase 테스트 모드 만료일(2026-05-11) 이후에는 Realtime Database → 규칙 탭에서 아래와 같이 수정 필요
> ```json
> { "rules": { ".read": true, ".write": true } }
> ```

## 블루투스 스캐너 사용 시 주의사항

- 블루투스와 2.4GHz 와이파이는 주파수 간섭이 발생할 수 있습니다 → **5GHz 와이파이** 사용 권장
- 스캐너와 컴퓨터 거리를 가깝게 유지하세요
- 첫 번째 스캔에서 소리가 약간 늦을 수 있습니다 (오디오 초기화) — 이후 스캔부터 정상
- 유선 USB 스캐너가 가장 빠르고 안정적입니다
